    Agent mode

    Assign work to four agents at once.

    Each session is pinned to a purpose and isolated in its own git worktree — agents never step on each other's diffs.

    agent · sessions 3 running

    OC flaky-tests

    Isolated worktrees

    Every session gets its own branch and working copy. Four agents can ship four features at the same time.

    Session purposes

    Pin a session to refactoring, bug-fixing or docs — the context and prompts follow the purpose.

    Context & plugin manager

    Curate exactly what each agent sees, and manage plugins per session from one panel.
